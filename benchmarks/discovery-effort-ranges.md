# Discovery Effort Ranges — Cross-Platform Benchmarks

Reference ranges for discovery phase activities. Applies regardless of technology stack. Update with actuals via Phase 5.

---

## Technical Discovery (Backend Tab equivalent)

| Task | Effort Range | Notes |
|------|-------------|-------|
| Requirements Workshops (per workshop cycle) | 16-32 hours | Budget 2x duration: prep + facilitation + synthesis |
| Stakeholder Interviews (per stakeholder group) | 4-8 hours | Includes scheduling, interview, and write-up |
| Technical Architecture Review | 24-48 hours | Current-state assessment, gap analysis, recommendations |
| Integration Landscape Mapping | 16-32 hours | Inventory all integrations, classify by tier, document data flows |
| Technology Evaluation & PoC | 24-60 hours | Research, spike, prototype, and documented recommendation |
| Content Architecture Analysis | 16-32 hours | Content types, taxonomy, editorial workflows, governance model |
| Performance & Security Audit | 16-40 hours | Core Web Vitals, Lighthouse, security headers, vulnerability scan |
| Infrastructure & Hosting Assessment | 8-16 hours | Existing environment review, hosting requirements, scalability needs |

## UX/Design Discovery (Frontend Tab equivalent)

| Task | Effort Range | Notes |
|------|-------------|-------|
| UX Research (interviews, personas, journey maps) | 24-48 hours | Recruiting, sessions, synthesis, persona/journey documentation |
| Information Architecture (sitemap, navigation) | 16-32 hours | Card sorting, sitemap, navigation structure, URL strategy |
| Wireframing & Prototyping | 40-80 hours | Low-to-mid fidelity wireframes for key templates and user flows |
| Design System Audit | 8-16 hours | Existing brand/component inventory, consistency assessment |
| Accessibility Assessment (WCAG audit) | 8-16 hours | Automated scan + manual review against WCAG 2.1 AA |
| Competitive/Comparable Site Analysis | 8-16 hours | 3-5 comparators, feature inventory, UX pattern benchmarking |

## Effort Scaling by Project Complexity

| Project Size | Total Discovery Budget | Indicators |
|-------------|----------------------|------------|
| Small | 80-160 hours | Single workstream, clear requirements, known tech stack, < 5 stakeholders |
| Medium | 160-320 hours | 2-3 workstreams, some ambiguity, integration mapping required, 5-15 stakeholders |
| Large | 320-500+ hours | Multi-workstream, complex integrations, migration involved, 15+ stakeholders, RFP-driven |

---

## Notes

- Discovery effort scales with project complexity: small projects 80-160h, medium 160-320h, large 320-500h+.
- Workshops must be budgeted at 2x their scheduled duration to account for preparation, facilitation, and post-session synthesis.
- Travel is budgeted separately from activity hours and varies by number of trips and location.
- These ranges are tech-agnostic and apply to Drupal, WordPress, Next.js, and custom builds equally.
- Confidence scoring (Conf 1-6) still applies when discovery tasks appear as line items in the estimate; use Conf 4-5 for most discovery tasks unless scope is explicitly bounded.
